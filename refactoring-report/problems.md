# Immediate problems to fix in `YoutubeService`

1. **Video/short URLs resolve to the literal word “watch”.** `_candidate_channel_params` only inspects URL path segments and never looks at query parameters, so a common share link like `https://www.youtube.com/watch?v=dQw4w9WgXcQ` yields the fallback identifier `"watch"`. That leads `_resolve_channel` to exhaust every lookup attempt and finally raise `YoutubeChannelNotFound` despite the channel existing.【F:backend/app/services/youtube.py†L101-L141】【F:backend/app/services/youtube.py†L192-L222】
2. **Channel feed URLs are unsupported.** Links such as `https://www.youtube.com/feeds/videos.xml?channel_id=UC…` or `?user=` are popular in documentation and RSS setups, but the helper never inspects query parameters, again collapsing to the string `"feeds"` and returning the wrong channel. This causes unnecessary API calls and eventual failure for otherwise valid identifiers.【F:backend/app/services/youtube.py†L101-L141】
3. **Malformed JSON from the API crashes the service.** `_request_json` assumes `response.json()` will succeed whenever the content type begins with `application/json`. If the body is truncated or empty (e.g. due to network proxies), `response.json()` raises a `ValueError`, which propagates past the YouTube-specific exception handling and surfaces as a 500.【F:backend/app/services/youtube.py†L323-L349】
4. **`max_results=0` still issues a request for one video.** `_fetch_videos` coerces the requested count with `max(1, max_results)`, so callers trying to perform a no-op validation or reuse the API key without fetching data cannot suppress item retrieval. This silently contradicts the method contract and may increase quota usage.【F:backend/app/services/youtube.py†L269-L283】
5. **No pagination when asking for more than 50 videos.** The Data API supports paging through playlist items via `nextPageToken`, but the implementation truncates results to a single page even if `max_results` is higher. Consumers requesting large batches receive fewer items than expected, and the wasted quota hampers reliability.【F:backend/app/services/youtube.py†L269-L321】

Addressing these defects will reduce user-facing errors, protect API quota, and make the service resilient to real-world URL formats.
